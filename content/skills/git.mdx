Left to its own devices, Claude Code writes commit messages like "updated files and fixed
some issues" and, worse, tags them with "Generated with Claude Code." Neither belongs in a
git history that's supposed to explain *why* things changed. Getting clean commits is a
matter of a clear standard.

## The job

Turn a diff into a well-formed [Conventional Commit](https://www.conventionalcommits.org/)
and a reviewable PR — with no AI-attribution noise.

## How to do it without a kit

**1. Give it the standard.** A [reusable skill](/guides/how-to-write-a-claude-code-skill)
with these rules covers it:

> From the staged diff, write a Conventional Commit: `type(scope): subject` under 72
> chars, where type is feat/fix/docs/refactor/test/chore. Add a body only if the change
> needs a *why*. Never add "Generated by" or "Co-authored-by: Claude" trailers.

**2. Enforce it with a hook.** A prompt can be forgotten; a
[hook](/guides/claude-code-hooks-examples) can't. Our free set includes an
`enforce-conventional-commits` hook that rejects non-conforming messages and AI-attribution
trailers automatically — the reliable way to keep a history clean.

**3. For PRs**, `gh` does the mechanical part: `gh pr create --fill` drafts from your
commits; ask Claude to write the description from the diff.

## What the kit's skill adds

AgentKit's `git` skill (`/ak:git`) wraps the whole flow — commit, push, PR creation, PR

## What it actually printed

We ran it on a deliberately messy tree — **four mixed concerns, and one planted secret**: a
fake OpenAI key, shape `sk-` + 32 chars, in a new file. The skill's own description promises
*"Auto-splits commits by type/scope. Security scans for secrets."* We tested both halves.

**The splitting worked, and it is genuinely good:**

````
e52a267 chore(package): add probe script
116bf9f docs(codebase-summary): add complete codebase documentation
d6cd17b docs(agentkit-facts): add probe comment to docs
97c13cb fix(source.config): tighten readingTime schema with positive constraint
````

Four concerns in, four conventional commits out, split on type *and* scope — it even separated
the 232-line new doc from a two-line edit to a *different* doc. That is the right call and not an
obvious one.

**The secret scan did not catch the secret.** Here is the check, verbatim from the skill's own
`SKILL.md:70`, and what happened when we staged the planted file and ran it:

````
$ git add src/lib/probe-config.ts
$ git diff --cached | grep -iE "(api[_-]?key|token|password|secret|credential)"
$ echo $?
1        # no match — the secret passed the gate
````

The file it missed contained `export const KEY = "sk-[REDACTED]..."`.

**It greps for keyword *names*, not secret *shapes*.** `sk-` matches none of `api_key`, `token`,
`password`, `secret`, or `credential` — and the variable is called `KEY`, which `api[_-]?key`
does not match either. A real-shaped key, correctly staged, in the diff it was scanning, and the
gate reported clean.

**The commit history it produced was clean. That is the problem.** A tidy, conventional,
well-scoped history is exactly what makes a review feel finished — and this one would have
carried a live key into a public repo with four beautifully-formatted commits wrapped around it.

> **Correction, 2026-07-13.** An earlier version of this page said the secret scan was one of
> "the parts most worth having." We had not run it. Having now run it, that was wrong, and this
> is the correction rather than a quiet edit.

## When NOT to reach for it

- **As your only secret gate.** It is not one. Use `gitleaks`, `trufflehog`, or a
  [pre-write hook](/guides/claude-code-hooks-examples) that blocks the value before it ever
  reaches a file.
- **When the split is obvious.** One concern, one commit. The value is on a messy tree.
- **On a shared branch, with `cp`.** `cp` pushes. Know that before you type it.

## Combining it

- **Before you commit:** [`/ak:code-review`](/skills/code-review) on the diff, then `/ak:git`.
  Splitting a mess into four clean commits does not make the mess correct — it makes it look
  reviewed.
- **The secret gate belongs earlier:** a
  [pre-write hook](/guides/claude-code-hooks-examples) blocks a credential before it is typed into
  a file. That is a real gate; this skill's grep for five English words is not one.
- **Shipping:** `/ak:git` for the commits → `/ak:ship` for the PR. Know that `cp` pushes before you
  type it.

## Buy if / Don't buy if

- **Worth it if** you often face a messy tree and want it split into clean, well-scoped
  conventional commits — that part is real, and it is better than most people do by hand.
- **Skip it if** you want the secret scan. Do not rely on it; see above. And if you only want
  clean commit *messages*, the [free hook](/guides/claude-code-hooks-examples) enforces that with
  nothing to buy.
