Sometimes the task lives in a browser, not the terminal — verifying a UI actually works,
grabbing a screenshot, filling a form, scraping a page. Claude Code can drive a real
browser for all of this. The free path is Playwright; the skill adds structure for longer,
self-verifying sessions.

## The job

Have Claude operate a real browser to test, screenshot, scrape, or automate a flow — and
confirm the result rather than assume it.

## How to do it without a kit

**Playwright is the free workhorse.** It's scriptable, headless-capable, and Claude writes
and runs it well:

```bash
npx playwright screenshot https://example.com shot.png
```

For interactive automation, have Claude write a Playwright script for the flow (navigate,
click, fill, assert) and *run it* — then read the result. The key discipline: make it
**verify**, not assume. "Click submit and confirm the success message appears," not just
"click submit."

For scraping, prefer a real API if one exists; use browser automation only when the data
is genuinely behind rendering. Respect robots and rate limits.

If you need the browser to use your real logged-in session (cookies, accounts), that's a
different setup — a profile-attached browser — and worth being deliberate about for
security.

## What the kit's skill adds

AgentKit's `agent-browser` skill (`/ak:agent-browser`) is built for longer autonomous
sessions with self-verification — accessibility-tree page snapshots, video recording, and
cloud-browser support (Browserbase) — beyond one-off Playwright scripts.

```bash
/ak:agent-browser "log in and verify the dashboard loads"
```

*(Legacy prefix: `/ck:agent-browser`.)*

You can read that, and you can click `@e3` without a selector. Compared with a raw HTML dump it

*"~280 chars/snapshot (vs 8K+ for Playwright MCP)"*. That is a number, so we counted:


**Both are larger than the 8K+ it attributes to the tool it is beating.** The smaller of the two

claimed figure. The homepage is 64×.

> **Note, added at publication.** `/skills/cook` was a near-empty stub on the day we measured it.
> It is [a written page now](/skills/cook) — we wrote it in the same batch of work that published
> this one. The number above is what the snapshot cost on the stub, and we are leaving it as

**The format is the product; the number is marketing.** Everything the snapshot actually gives
you — semantic structure, stable refs, no CSS noise — is real, and none of it depended on being
280 characters. But `~280 chars/snapshot` is a claim about your *context budget*, and context
budget is the entire reason an agent-oriented browser tool exists. If you are planning a long
autonomous session around that figure, you are planning around a number that is off by a factor
of 50 on a small page.

**Budget for what you measure on your own pages.** We only know because we ran `wc -c`.

## When NOT to reach for it

- **When you need your real Chrome profile.** This drives a fresh browser — no cookies, no
  logged-in session. That is deliberate (it is reproducible), and it is a hard limit.
- **When you want assertions, not exploration.** This drives and reports;
  [`/ak:web-testing`](/skills/web-testing) is what you want if the output should be pass/fail.
- **In a tight script, without a timeout.** Our first attempt hung for four minutes:
  `agent-browser open` holds the browser and does not return. It works with an explicit timeout
  and a separate `snapshot` call — but the failure mode is a hang, not an error.

## Combining it

- **Exploratory QA:** `agent-browser` to drive and look → [`/ak:web-testing`](/skills/web-testing)
  to turn whatever you found into an assertion that fails next time. Driving is not testing until
  something can go red.
- **After a UI change:** [`/ak:ui-ux-pro-max`](/skills/ui-ux-pro-max) reviews the code; this drives
  the rendered result. The two disagree more often than you would expect, and the browser wins.
- **When a page renders wrong:** [`/ak:debug`](/skills/debug). A snapshot shows you *that* the tree
  is wrong; it will not tell you why.

## Buy if / Don't buy if

- **Worth it if** you run long autonomous browser sessions and want an accessibility-tree
  snapshot with stable refs instead of parsing a DOM dump.
- **Skip it if** you need occasional screenshots or a test — Playwright plus Claude Code is
  free and plenty. And do not buy it for the context-budget claim; we measured it and it does
  not hold.
