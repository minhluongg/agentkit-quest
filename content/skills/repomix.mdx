Sometimes you want to hand an entire codebase to an LLM at once — for a security audit, to
understand an unfamiliar library, or to onboard. [Repomix](https://github.com/yamadashy/repomix)
packs a whole repo into a single structured file for exactly that. It's free and open
source; here's how to use it and what the skill adds.

## The job

Turn a repository into one AI-consumable file — XML, Markdown, or plain text — so a model
can reason over the whole thing in one context.

## How to do it without a kit

Repomix is a free npm tool. Run it directly:

```bash
npx repomix                          # pack the current repo
npx repomix --style markdown         # markdown output
npx repomix --include "src/**"       # scope it
```

The output is one file with the directory tree and every file's contents, formatted for an
LLM. Two things to get right:

**1. Scope it.** Packing `node_modules` wastes the whole context window. Use `.repomixignore`
(or `--include`) so you pack the code, not the dependencies.

**2. Pick the format for the consumer.** XML parses most reliably for models; Markdown reads
best for humans; plain text is smallest.

Then paste (or attach) the packed file into your LLM of choice. This is a genuinely useful
free tool for third-party library analysis and audits.

## What it actually printed

We packed this site's own `src/` — 57 files of Next.js app code:

```bash
npx repomix src --style markdown --compress
```

Real output (ANSI codes stripped, the absolute output path redacted — it was a home
directory, not yours):

````

📈 Top 5 Files by Token Count:

📊 Pack Summary:
────────────────
  Total Files: 57 files
 Total Tokens: 30,068 tokens
  Total Chars: 122,635 chars
       Output: [REDACTED]/repomix-src.md
     Security: ✔ No suspicious files detected
````

**Two generated JSON files are 57.6% of the pack.** `skills.generated.json` alone is 14,712
tokens — 48.9% — and it is a build artefact, not code anyone reads. Pack `src/` naively and
more than half your context budget goes to data the model does not need in order to reason
about the app. The fix is one `--ignore` glob, and you only know to reach for it **because the
tool ranked the files for you.**

That ranking is the thing a `cat`-everything-into-a-file approach never gives you. Neither is
the security scan: it checked all 57 files for credentials before writing the pack. This repo
does have a live token in a `.env` file — `src/` does not contain it, and repomix *confirms*
that rather than asking you to assume it.

## When NOT to reach for it

- **You only need part of the repo.** Packing everything to ask about one module is the
  expensive way to get a cheap answer. [`/ak:scout`](/skills/scout) explores without filling
  your context.
- **You are already in Claude Code, in the repo.** Claude can read the files directly. Packing
  is for getting a codebase *out* — into a different model, a web UI, an audit.
- **The repo is large.** 57 files was 30k tokens. A real application will blow any context
  window; scope it hard, or pick a different tool.
