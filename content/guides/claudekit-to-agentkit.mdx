If you bought or used **ClaudeKit** and can no longer find it, nothing was taken away —
it was **renamed to AgentKit**. Same product, same kits, new name. Here is precisely what
changed and the one thing you may need to update.

## What changed

<RenameDiagram />

The old domain `claudekit.cc` **301-redirects** to `agentkit.best`, and `docs.claudekit.cc`
moved with it.

The kits themselves — Engineer and Marketing — are the same products with the same
purpose. Your licence carries over; you did not lose access by the rename.

## What you actually need to do

For most people: **nothing urgent.** But two things are worth knowing.

<Steps>
  <Step>
    ### Update how you invoke it

    The commands you type changed prefix. Where you used to run:

    ```bash
    /ck:plan "add authentication"
    /ck:cook "implement the plan"
    ```

    you now run:

    ```bash
    /ak:plan "add authentication"
    /ak:cook "implement the plan"
    ```

    The CLI binary is `ak` instead of `ck`. If you have a shell alias or a script that
    calls `ck`, point it at `ak`.
  </Step>

  <Step>
    ### Let `ak migrate` do the move for you

    You do not have to migrate by hand. The CLI ships a command specifically for this, and
    it is **dry-run by default** — it writes nothing until you explicitly opt in:

    ```bash
    ak migrate                  # scan and show the plan. Writes nothing.
    ```

    It scans `~/.claude/`, `~/.claudekit/`, and the current directory, classifies what it
    finds, and prints the plan it *would* apply. Run on a machine with a real ClaudeKit
    install, ours reported:

    ```
    [i] Legacy ClaudeKit kit preflight: 4 finding(s)
      - claude-code global  engineer  confidence=high  hook-risk=none  artifacts=64
        root: C:\Users\Admin\.claude
      - claude-code project engineer  confidence=high  hook-risk=none  artifacts=84
        root: ...\agentkit-quest-fe\.claude
      - claude-code project unknown   confidence=low   hook-risk=manual-review  artifacts=1

    [i] Legacy plan 2f492fd97239: archive=149 neutralize=149 install=3 preserve=1 conflict=0
      archive: C:\Users\Admin\.agentkit\archives\legacy-kit-migration\20260712T183114Z
      - preserve ...\.claude\settings.json (low-confidence or unknown legacy artifact
        requires manual review)
    ```

    Read that plan before you apply it. **`archive=149 neutralize=149` means it will move
    and deactivate 149 files** — including, in our case, a project-local `.claude/` we had
    hand-tuned. It archives rather than deletes, and it kept `settings.json` back for manual
    review rather than guessing. But "archive" still means "your current setup stops being
    the live one".

    When the plan looks right:

    ```bash
    ak migrate --from=ck --dry-run=false --yes
    ```

    And if it goes wrong, there is a way back:

    ```bash
    ak migrate rollback
    ```
  </Step>
</Steps>

<Callout type="tip" title="Or just leave ClaudeKit alone">
Both CLIs coexist. After installing `ak` on a machine with `ck` (claudekit-cli 4.5.0), `ck`
kept working exactly as before — we checked. You are allowed to install `ak`, look around,
and migrate later. `ak doctor` will nag you about a "ck shim collision" until you resolve it,
but nothing is broken by having both.
</Callout>

<Callout type="warn" title="A quirk worth knowing">
Some **internal** references in the shipped kit still say `ck:` even after the rename —
the migration of every string is ongoing. If you see `ck:` inside the kit's own files
while the docs say `ak:`, that is the rename catching up, not a mistake on your end. The
`/ak:` prefix is the current, correct one to type.
</Callout>

## Is ClaudeKit gone? Should I be worried?

No. This is a rename and a slight repositioning (the "AgentKit" name reflects that it is
about AI *agents*, and there is now a desktop app in the works), not a shutdown. A small
vendor renaming once is normal. Your purchase stands.

If you are a *new* reader who just found this and does not have the kit yet, the rename
is not something you need to think about — you will only ever see "AgentKit." Start with
[what AgentKit is](/guides/what-is-agentkit), or, if you are weighing the purchase, the
[honest worth-it comparison](/guides/agentkit-vs-free-alternatives).

## Quick reference

- **Old name → new name:** ClaudeKit → AgentKit
- **Old CLI → new CLI:** `ck` → `ak`
- **Old prefix → new prefix:** `/ck:` → `/ak:`
- **Old site → new site:** `claudekit.cc` → `agentkit.best` (auto-redirects)
- **Your licence:** carries over, nothing to repurchase
