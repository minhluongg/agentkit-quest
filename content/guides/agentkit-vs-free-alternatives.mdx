## Where the free option clearly wins

**If you need one or two specific capabilities, do not buy a kit.** Say you want a good
code-review command and a test-runner. Copy those two from awesome-claude-code-toolkit,
or write them — a `SKILL.md` is a title, a description, and a body of instructions. You
will spend an hour and pay nothing. Buying a 91-skill kit for two skills is like buying
a toolbox for one screwdriver.

**If assembling the setup is the part you enjoy,** the free primitives are the whole
game. Skills, subagents and hooks are all free, first-party Claude Code features. A kit
does not unlock anything the platform withholds — everything AgentKit does, you *can*
build.

That is the honest free case, and for a lot of developers it is the right answer. Now the
other side.

## What the paid kit actually gives you

Having used AgentKit's workflow to build real things, the value is not the 91 skills. It
is three things that a folder of 135 loose agents does not have:

**1. Coherence — the pieces hand off to each other.** The planning skill produces a plan
file in a specific shape; the implementation skill reads exactly that shape; the review
step knows where the plan is. With a collection of independently-authored free agents,
you get 135 good pieces that do not share a contract, and you spend your time being the
integration layer between them. The paid kit's real product is the *wiring*, not the
parts.

**2. The subagents are genuinely useful, and pre-configured.** Claude Code subagents run
in their own context and return a summary — the value is context isolation (see
[skills vs subagents](/guides/skills-vs-commands-vs-subagents)). AgentKit ships a
code-reviewer, a researcher, a red-team reviewer and others already scoped to the right
tools. In practice these catch things: a review subagent flags bugs the main thread
wrote; a red-team subagent argues against a plan before you commit to it. You can build
these yourself — but "already built and tuned" is the thing you are paying for.

**3. It is maintained.** The kit is on version 2.20.0 and ships regular releases. The
large free awesome-lists are excellent snapshots that go stale — Claude Code changed how
commands and skills work in 2026, and a maintained kit tracks that while a starred-but-
abandoned repo does not.

## The honest catch

Two things the sales pages will not lead with:

- **The kit is a small vendor.** It is a one-person operation that has already renamed
  itself once (ClaudeKit → AgentKit). That is fine for a $99 one-time purchase, but do
  not expect enterprise-grade longevity guarantees.
- **You still have to learn it.** A kit of 91 skills has its own conventions to absorb.
  It is faster than building from scratch, but it is not zero-effort — budget an
  afternoon to get productive.
